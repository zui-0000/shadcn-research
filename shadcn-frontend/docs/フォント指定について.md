# フォント指定について

## 概要

Tailwind CSS プロジェクトでは、フォントの**読み込み**と**Tailwind のユーティリティ（`font-sans` など）への登録**を分けて考える。本プロジェクトは Tailwind CSS v4 を採用しており、グローバル CSS（例: `src/styles/index.css`）の `@import` と `@theme` でまとめて扱うのが一般的である。

---

## 指定する場所の整理

| やりたいこと | 指定する場所 |
|---|---|
| フォントファイルの読み込み | グローバル CSS の `@import`（例: `@fontsource-variable/geist`）、または `index.html` の `<link>`（Google Fonts 等） |
| `font-sans` などのクラス名とファミリー名の対応 | Tailwind v4 では `@theme` 内の `--font-*`。v3 や `tailwind.config` を使う場合は `theme.extend.fontFamily` |
| アプリ全体のデフォルトフォント | `html` または `body` に `@apply font-sans`、またはルート要素に `className="font-sans"` |

フォントの実体を読み込まずに `font-family` だけ定義しても、ブラウザにフォントが入っていなければ期待どおり表示されない。必ず読み込みと `@theme`（または config）の両方を揃える。

---

## Tailwind CSS v4（CSS ファースト）の例

グローバル CSS に次のような記述を置く。

```css
@import "tailwindcss";
@import "@fontsource-variable/geist";

@theme {
  --font-sans: "Geist Variable", sans-serif;
}

@layer base {
  html {
    @apply font-sans;
  }
  body {
    @apply antialiased;
  }
}
```

- `@import "@fontsource-variable/geist"` … npm パッケージ経由で可変フォントをバンドルする例
- `@theme` の `--font-sans` … `font-sans` ユーティリティに紐づく
- `html { @apply font-sans }` … 未指定の要素もデフォルトで sans 系になる

shadcn/ui テーマと併用する場合は、既存の `@theme inline` ブロック内に `--font-sans` を書く構成もある。プロジェクトの `index.css` の実際の構成に合わせる。

---

## `tailwind.config` で指定する場合（参考）

Tailwind v3 形式や、v4 で `@config` により `tailwind.config.ts` を読み込んでいる場合は、次のように `fontFamily` を拡張できる。

```typescript
export default {
  theme: {
    extend: {
      fontFamily: {
        sans: ['"Geist Variable"', "sans-serif"],
      },
    },
  },
};
```

フォントファイル自体の読み込みは、別途 CSS の `@import` または HTML の `<link>` で行う必要がある。

## 見出しだけ別フォントにする（`@theme` で `--font-*` を追加）

Tailwind v4 では、`@theme` に `--font-` で始まる変数を追加すると、対応する `font-*` クラスが生成される。

### CSS の例

```css
@import "tailwindcss";
@import "@fontsource-variable/geist";
/* 見出し用フォントを別パッケージで入れる例 */
/* @import "@fontsource-variable/playfair-display"; */

@theme {
  --font-sans: "Geist Variable", sans-serif;
  --font-heading: "Playfair Display Variable", Georgia, serif;
}
```

`--font-heading` を定義すると、コンポーネント側で **`font-heading`** クラスが使える。

### 変数名とクラスの対応

| `@theme` の変数 | 生成されるユーティリティ |
|---|---|
| `--font-sans` | `font-sans` |
| `--font-serif` | `font-serif` |
| `--font-mono` | `font-mono` |
| `--font-heading`（任意の名前） | `font-heading` |

命名は「`--font-` + 識別子」が分かりやすい。実際の `font-family` 文字列は、読み込んだフォントの PostScript 名・CSS 名と一致させる。

### コンポーネントでの使い分け

```tsx
<h1 className="font-heading text-3xl font-bold">ページタイトル</h1>
<p className="font-sans text-base">本文テキスト</p>
```

### 全見出しを一括で揃えたい場合

`@layer base` で要素セレクタに `@apply` する方法もある。

```css
@layer base {
  html {
    @apply font-sans;
  }
  h1,
  h2,
  h3 {
    @apply font-heading;
  }
}
```

個別に制御したい場合はコンポーネントで `font-heading` を付け、一括なら上記の base レイヤーが向いている。

---

## まとめ

- フォントは**読み込み**（`@import` / `<link>`）と**テーマ登録**（`@theme` の `--font-*` または `tailwind.config`）のセットで指定する
- v4 では `@theme` が中心で、`font-sans` などは CSS 変数から自動生成される
- 見出し用に `--font-heading` のような別名を足し、`font-heading` で使い分けることができる
